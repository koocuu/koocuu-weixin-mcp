# 产品文档：koocuu-weixin-mcp

## 要解决的问题

让 AI 客户端（Claude / Codex / ChatGPT）通过 Remote MCP 管理微信公众号：写稿、上传素材、草稿、菜单、二维码，以及受控发布。需要稳定公网入口与固定出口 IP，以满足微信 API IP 白名单。

## 目标用户

公众号运营者本人（单账号、自用）。不是多租户 SaaS。

## 核心场景

1. 在 Claude / Codex 中连接 `https://weixin.koocuu.com/api/mcp`，完成一次 OAuth 授权（授权页粘贴 `MCP_BEARER_TOKEN`）。
2. 用 MCP 工具生成文章 HTML、上传封面/正文图、创建与更新草稿。
3. 在确认后发布草稿（默认关闭，需显式开启并二次确认）。
4. 接收微信服务器回调校验与可选明文自动回复。

## 功能范围

- Markdown/HTML → 微信图文 HTML
- 临时/永久图片素材上传
- 草稿 CRUD、菜单读写、二维码 ticket
- 出站 IP 探测（经 SCF 中继时返回固定出口 IP）
- 受控 freepublish（`WECHAT_ENABLE_PUBLISH` + `confirm` + `dryRun`）
- OAuth（MCP 客户端拿 bearer，非多用户账号体系）

## 非目标

- 群发、多公众号租户、复杂权限体系
- 依赖本机长期开机的生产方案（本地 Tunnel 仅作开发/过渡）

## 架构流程（混合，已上线）

```text
Claude / Codex / Cursor
        │
https://weixin.koocuu.com → Vercel（MCP + OAuth + 回调）
        │
        │ WECHAT_RELAY_URL
腾讯云 SCF 广州 /api/wechat-relay（固定出口 129.204.3.249）
        │
微信公众号 API
```

说明：Claude.ai 不宜直连广州 SCF（延迟/SSE）；Vercel 作入口、SCF 作微信出口。Upstash Redis 可选，用于跨冷启动保存 OAuth/token。

## 当前部署状态（摘要）

- 公网 MCP：`https://weixin.koocuu.com/api/mcp`（Vercel）
- 备用：`https://koocuu-weixin.vercel.app/api/mcp`
- 微信中继：SCF `…ap-guangzhou.tencentscf.com/api/wechat-relay`
- 固定出口 IP：`129.204.3.249`（需在微信 API IP 白名单）
- 微信回调：`https://weixin.koocuu.com/api/wechat/callback`（同打到 Vercel）

## 验收标准

1. Claude 自定义连接器可用 `https://weixin.koocuu.com/api/mcp` 完成 OAuth；对话中可调用工具。
2. `wechat_get_outbound_ip` 返回 `129.204.3.249`。
3. 微信回调 URL 校验通过。
4. 发布类工具默认不可执行；开启后仍需 `confirm` + `dryRun:false`。
