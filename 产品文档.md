# 产品文档：koocuu-weixin-mcp

## 要解决的问题

让 AI 客户端（Claude / Codex / ChatGPT）通过 Remote MCP 管理微信公众号：写稿、上传素材、草稿、菜单、二维码，以及受控发布。需要稳定公网入口与固定出口 IP，以满足微信 API IP 白名单。

## 目标用户

公众号运营者本人（单账号、自用）。不是多租户 SaaS。

## 核心场景

1. 在 Claude / Codex 中连接 Vercel MCP 入口，完成一次 OAuth 授权（授权页粘贴 `MCP_BEARER_TOKEN`）。
2. 用 MCP 工具生成文章 HTML、上传封面/正文图、创建与更新草稿。
3. 在确认后发布草稿（默认关闭，需显式开启并二次确认）。
4. 接收微信服务器回调校验与可选明文自动回复。

## 功能范围

- Markdown/HTML → 微信图文 HTML
- 临时/永久图片素材上传
- 草稿 CRUD、菜单读写、二维码 ticket
- 出站 IP 探测（混合模式下探测 SCF 固定出口）
- 受控 freepublish（`WECHAT_ENABLE_PUBLISH` + `confirm` + `dryRun`）
- OAuth（MCP 客户端拿 bearer，非多用户账号体系）

## 非目标

- 群发、多公众号租户、复杂权限体系
- 依赖本机长期开机的生产方案（本地 Tunnel 仅作开发/过渡）

## 架构流程（混合）

```text
Claude / Codex / Cursor
        │
Vercel（MCP + OAuth，低延迟入口）
        │  WECHAT_RELAY_URL → /api/wechat-relay
腾讯云 SCF 广州（固定出口 IP）
        │
微信公众号 API

微信服务器回调：
微信 → weixin.koocuu.com（Cloudflare Worker）→ SCF /api/wechat/callback
```

说明：Claude.ai 直连广州 SCF 会因跨境延迟/SSE 失败；Vercel 作入口、SCF 作微信出口，同时满足连接性与 IP 白名单。

- 代码：已支持 `WECHAT_RELAY_*`；需完成 SCF 镜像更新 + Vercel 环境变量后验收

## 验收标准

1. Claude 自定义连接器可用 Vercel MCP 完成 OAuth 并调用工具。
2. `wechat_get_outbound_ip` 返回 SCF 固定出口 IP（非 Vercel 动态 IP）。
3. 微信回调 URL `https://weixin.koocuu.com/api/wechat/callback` 校验通过。
4. 发布类工具在默认配置下不可执行；开启后仍需 `confirm` + `dryRun:false`。
